🚀 Phase 1 – From SQL Thinking to PySpark Thinking
🧠 What I Did in This Phase

In this phase, I basically shifted my mindset from writing SQL queries to working with PySpark.

At first, I thought it would just be about learning new syntax… but it turned out to be more about changing how I think about data processing.

Instead of writing one big query like in SQL, I started breaking things down into step-by-step transformations using PySpark.

🧩 What I Learned

I worked with PySpark DataFrames and practiced some core operations like:

Selecting specific columns from datasets
Filtering data based on conditions
Grouping data and performing aggregations
Checking outputs at every step to understand what's happening

This helped me get comfortable with how structured data is handled in PySpark.

🔁 How I Connected SQL with PySpark

Instead of memorizing syntax, I tried to map SQL concepts to PySpark, which made things much easier:

SQL SELECT → PySpark column selection
SQL WHERE → PySpark filtering
SQL GROUP BY → PySpark grouping + aggregation

This approach really helped me bridge the gap between what I already knew and what I was learning.

🛠️ What I Practiced

To get hands-on, I worked on small tasks like:

Exploring datasets to understand their structure
Applying filters (like age-based conditions)
Extracting meaningful subsets of data
Performing grouped analysis

These exercises gave me more confidence in working with real-world data.

⚡ Challenges I Faced

Honestly, there were a few struggles:

Switching from SQL mindset to PySpark syntax
Understanding why grouping works differently in PySpark
Getting used to writing logic step-by-step instead of one query

But once I practiced more, it started making sense.

💡 What Changed in My Thinking

This phase changed how I look at data processing:

PySpark is not just “SQL in another form”
It gives more control and flexibility
Breaking problems into steps actually makes things clearer
Concepts matter more than syntax
🎯 Outcome

By the end of this phase, I can:

Perform basic transformations using PySpark confidently
Easily relate SQL logic to PySpark operations
Approach data problems in a more structured way
